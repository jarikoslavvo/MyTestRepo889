# Начинаю изучение Git

 Git - распределенная система контроля версий , которая позволяет разработчику , или команде разработчика полностью контролировать репозиторий, а также
 связывать свой локальный репозиторий с сервисом хостинга репозиториев - GitHub. 

## Почему стоит использовать Гит? 

 * - система контроля версий , откат в случае необходимости
 * - возможность разработки новых функций приложения путем создания новых веток приложения , которые не влияют на основную ветку.
 * - командная работа всех разработчиков проекта с единым централизованным репозиторием на ГитХаб.

 Команда `git config --global user.name <name>` - нужна для того , чтобы указывать в метаданных коммита , кто собственно закомитил.
 Команда `git config --global user.email <email>` - добавляет в мета-данные коммита почту коммитящего.

 опция `--global` добавляет глобальные настройки Гит для всех ваших локальных репозиториев (например имя и почта).

 Команда `git config --list` позволяет проверить ваши настройки конфигурационного файла Гит.

## Инициализация репозитория и начало работы с Git

 Чтобы инициализировать локальный репозиторий Гит , следует ввести эту команду : 

> git init

 Теперь в вашей директории есть скрытая папка .git , которая хранит все обьекты Гит.

## Области Гит.

| Стадия (Stage) | Команда | Описание | Статус файла |
| :--- | :--- | :--- | :--- |
| **Working Directory** | `git status` | Рабочая директория, где вы редактируете файлы. Файл изменен, но еще не подготовлен. | Изменен (Modified) / Не отслеживается (Untracked) |
| **Staging Area (Index)** | `git add <файл>` | Область подготовленных файлов. Файл готов к коммиту. | Подготовлен к коммиту (Staged) |
| **Local Repository** | `git commit -m "msg"` | Локальное хранилище. Изменения сохранены в истории вашего компьютера. | Закоммичен (Committed) |
| **Remote Repository** | `git push` | Удаленный сервер (GitHub, GitLab). Файл доступен другим участникам. | Отправлен (Pushed) |

## Строение коммита Git:

* - SHA-1 хэш код (уникальный идентификатор)
* - описание коммита
* - имя пользователя , который сделал коммит
* - email пользователя , который сделал коммит
* - родительский(е) коммит(ы) SHA-1
* - дерево SHA-1

## Что такое HEAD? 

**HEAD** - специальный указатель , который указывает на ту версию проекта , которую мы может наблюдать в нашей рабочей директории. 
> Грубо говоря , это то , где мы сейчас.

HEAD ссылается на текущую ветку , в то время как ветка ссылается на последний коммит .

**НО** , если мы сделаем `git checkout commit_name123` , то **HEAD** будет ссылаться сразу на тот коммит , минуя ветку.

## Основные команды Git

* `git status` эта команда может вывести статус вашего проекта , в какой вы ветке , какие файлы в индексе , а какие нет и т.д
* `git add <file>` эта команда позволяет добавлять нужные файлы на **stading area** , или же **index**
* `git commit -m "message"` эта команда позволяет закоммитить изменения с указанием комментария 
* `git log` эта команда выводит полную историю изменений , а также все коммиты с хэшами и комментариями
* `git checkout <hash>` эта команда позволяет переключиться на другой коммит (другую версию проекта)
* `git checkout <branch name>` эта команда переключает вас на другую определенную ветку

# Ветки в Git 

Ветки - отдельные версии развития вашего проекта , в которых вы можете разрабатывать новый функционал вашего приложения,
при этом изолируя ее от ветки master/main.

**Как создать ветку?**

* `git branch "branch_name"` - позволяет создать новую ветку.
* `git checkout "branch_name"` - позволяет переключиться на новую ветку.
* `git checkout -b "branch_name"` - делает две эти команды сразу в одной.
* `git branch -m "new_branch_name"` - переименовывает **ТЕКУЩУЮ** ветку.
* `git branch -d "branch_name"` - удаление ветки (важно быть в другой ветке , чтобы удалить).

# Слияние веток

* `git merge "имя feature-ветки"` - совершает слияние feature ветки с receiving веткой (текущей). 

> Эту команду нужно совершать из той ветки , куда вы хотите добавить новый функционал.
